## Design decisions

1. **Clock = timestamp** (`mode=timestamp`), not block number. With 0.5 s blocks the block-based
   mode is awkward. The token and the Governor must share the clock; the Governor inherits it from
   the token (`GovernorVotes.clock()` reads `token().clock()`), so the override lives only in
   `GovToken`.
2. **Treasury asset in tests = MockERC20** (hermetic golden test). On Arc we swap it for real
   testnet USDC (`0x3600...0000`, 6 decimals).

## Contracts (deliberately vulnerable — BONK class)

- `GovToken` — ERC20Votes, timestamp clock, supply minted to the initialHolder.
- `Treasury` — Ownable(Governor), `withdraw` for the owner only; **no timelock**.
- `DAOGovernor` — quorum **1%**, `proposalThreshold = 0` (anyone proposes), **no timelock**.
- `MockERC20` — stand-in treasury asset.

Three pillars of vulnerability: low quorum (1%) + zero proposal threshold + no defense window (timelock).
